# Subscription Overload Manager — PRD

## Original problem statement
Full-stack web application to help users manage, track and optimize recurring digital subscriptions (OTT, SaaS, cloud, productivity). Provides expense tracking, renewal alerts, usage-based optimization, budget limits, forecasts, and JWT-secured multi-device sync. Currency = INR.

## Tech (running)
- **Backend**: FastAPI + MongoDB (Motor) + APScheduler + Resend (email) + PyJWT + bcrypt
- **Frontend**: React 19 + Tailwind + shadcn + Recharts + sonner
- **Design**: Swiss minimalist — Cabinet Grotesk + IBM Plex Sans + Klein Blue accent (#002FA7)

## Parallel codebase
`/app/nodejs_backend/` — Node.js + Express + Mongoose equivalent (for user's academic submission / Render deployment). Not running; starts with `npm install && npm run dev`.

## Implemented (Apr 2026)
### Auth
- Register / Login / /me with JWT (30-day expiry)
- bcrypt-hashed passwords

### Subscriptions
- CRUD, auto-categorization (keyword-based), per-user scoping
- `mark used` endpoint to track usage frequency
- Billing cycles: monthly / yearly / weekly (normalized to monthly for analytics)

### Analytics
- `/api/analytics/summary` — active count, monthly/yearly totals, category breakdown, upcoming renewals, budget status
- `/api/analytics/forecast` — 6-month projection

### Recommendations (rule-based)
- Cancel (25+ days unused)
- Cancel (0 usage in 14+ days since add)
- Downgrade (>₹500/mo premium spend)
- Reminder (renewal in ≤3 days)

### Notifications
- In-app bell + slide-over panel
- Renewal reminders auto-generated by daily cron
- Email renewals via Resend (HTML template, INR)

### Budget
- Per-user monthly limit
- Over-budget pill on dashboard

### Frontend pages
- Auth (split-screen), Dashboard, Subscriptions (list + filters), Analytics (Pie/Bar/Line), Settings
